Event Handling

Understanding Event Handling in React

Handling events in React is similar to handling events on DOM elements, but with some syntactic differences. React events are named using camelCase instead of lowercase, and you pass a function as the event handler rather than a string.

In HTML, you might write: <button onclick="activateLasers()">Activate Lasers</button>

In React, it would be: <button onClick={activateLasers}>Activate Lasers</button>

Another difference is that in React you cannot return false to prevent default behavior. You must call preventDefault() explicitly.

Example: Basic Event Handling

function Button() {
  function handleClick() {
    alert('Button was clicked!');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

Example: Event Handling with Arrow Function

function ToggleButton() {
  const [isToggleOn, setIsToggleOn] = useState(true);

  return (
    <button onClick={() => setIsToggleOn(!isToggleOn)}>
      {isToggleOn ? 'ON' : 'OFF'}
    </button>
  );
}

Example: Handling Events with Parameters

function ActionButton() {
  const [message, setMessage] = useState('');

  const handleClick = (action) => {
    setMessage(`Action performed: ${action}`);
  };

  return (
    <div>
      <button onClick={() => handleClick('Save')}>Save</button>
      <button onClick={() => handleClick('Delete')}>Delete</button>
      <p>{message}</p>
    </div>
  );
}
