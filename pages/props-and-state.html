Props and State

Understanding Props and State

Props (short for properties) are how components talk to each other. They are passed from parent components to child components and are immutable - meaning they cannot be changed from within the component that receives them.

State is an object that holds data that may change over the lifetime of a component. Unlike props, state is managed within the component itself and can be updated using methods like setState in class components or useState in functional components with hooks.

Props are used to pass data down the component tree, while state is used to manage data that changes within a component.

Example: Using Props

function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Using the component with props
const element = <Welcome name="Sarah" />;

Example: Using State in Class Components

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={() => this.setState({count: this.state.count + 1})}>
          Increment
        </button>
      </div>
    );
  }
}

Example: Using State with Hooks (Functional Components)

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}
